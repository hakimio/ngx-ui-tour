Step Configuration

Each step can have the following properties.

A unique identifier for the step (optional)
The anchor to which the step will be attached
 The title of the tour step The content text of the tour step
Enable active element highlighting
 Backdrop configuration Step width, min-width and max-width configuration. Accepts CSS property "width" values, ie "250px" or "auto".
The route to navigate to before showing the tour step (optional)
Custom popover CSS class (optional)
 Enable to close the tour on outside click Disables scrolling when the UI tour is active. scrollContainer option can be used to specify affected scroll container. If scrollContainer is not set, disables scrolling on document element. The step index or stepId of the next step (optional) The step index or stepId of the previous step (optional)
Tour step position with respect to the anchor (optional).
If enabled, shows an arrow that points to the anchor element when running in ios mode. Does not apply in md mode. If active, stops automatic page scroll to tour anchor element
Enable to center active anchor element. "Safari" browser does not support this option.
Use smooth scroll when animations are enabled on OS level. Controlled by "Animate controls and elements inside windows" setting on Windows OS.
CSS selector or html element reference. Used to determine scroll container for "smooth scroll" and "disable page scrolling" functions. Defaults to document element if not set.
"Previous" button title
 "Next" button title "End" button title
Mark your step as "async" if anchor element is added to DOM with a delay.
 Mark your step as "optional" if it should be skipped when anchor element is not found. Step can not be marked both "optional" and "async". Delay between navigation to a different route and showing the tour step in ms. Should be set if the tour tries to show tour step before anchor element is registered or the page is scrolled all the way. Delay before showing tour step in ms.
Go to next step on anchor element click instead of "Next" button.
If your tour anchor is duplicated on purpose, you can choose to either register first or last anchor occurance instead of throwing an error. Enable to keep the tour open while the user navigates to a different page. Can cause the tour to get "stuck" if active anchor element is removed from the DOM after the navigation. The option is not related to navigation initiated by the UI tour and should not be enabled in most cases. Enable to use popover title for showing step title like in previous versions of the library. Title style can not be customized using step template when the option is enabled.
