There are a couple IMPORTANT limitations to note when using <tour-proxy-anchor>:

  • If the targeted element is inside <ng-template> or in a conditionally toggled container (@if, @switch, @for, etc), <tour-proxy-anchor> should also be placed in the same container. In general, it's a good idea to always place <tour-proxy-anchor> as close to the targeted element as possible.
  • touranchor--is-active CSS class will not be applied to the active anchor if the anchor was defined using <tour-proxy-anchor> component.
